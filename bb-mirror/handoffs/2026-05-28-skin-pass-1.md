Read-side strangler for BuddyBoss/bbPress forum threads. Forum URLs render
out of bb-mirror's own postgres + FPM pool at native-static speed; writes
still round-trip through BB REST.

## How to test

```bash
TOK=$(sudo grep -E 'set \$loothdev_token' \
  /etc/nginx/sites-available/dev.loothgroup.com.conf | \
  head -1 | grep -oE '"[^"]+"' | tr -d '"')
curl -s "https://dev.loothgroup.com/claim?t=$TOK" -c /tmp/bbjar -o /dev/null

# All three should 200
curl -s -b /tmp/bbjar -o /dev/null \
  -w "%{http_code}  %{size_download}b  /forums-poc/\n" \
  "https://dev.loothgroup.com/forums-poc/"
curl -s -b /tmp/bbjar -o /dev/null \
  -w "%{http_code}  %{size_download}b  /forums-poc/acoustic/\n" \
  "https://dev.loothgroup.com/forums-poc/acoustic/"
curl -s -b /tmp/bbjar -o /dev/null \
  -w "%{http_code}  %{size_download}b  /forums-poc/general/stripped-out-trussrod/\n" \
  "https://dev.loothgroup.com/forums-poc/general/stripped-out-trussrod/"

# Find topics with threaded replies for single-topic tests
sudo -u bb-mirror psql -d looth -c "
  SELECT t.slug AS topic_slug, f.slug AS forum_slug
    FROM forums.topic t JOIN forums.forum f ON f.id = t.forum_id
   WHERE t.id IN (SELECT DISTINCT topic_id FROM forums.reply
                  WHERE parent_reply_id IS NOT NULL)
   ORDER BY t.reply_count DESC LIMIT 5;"
```
