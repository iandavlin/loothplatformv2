# BB-mirror — Session Handoff (2026-05-28, render bugs fixed)

## What this project is

Read-side strangler for BuddyBoss/bbPress forum threads. Forum URLs render
out of bb-mirror's own postgres + FPM pool at native-static speed; writes
still round-trip through BB REST so notifications/moderation/mentions/
presence keep working unchanged.

Scope contract: [STRANGLER-COORDINATION.md §3f](../docs/STRANGLER-COORDINATION.md).
Storage: [§3i](../docs/STRANGLER-COORDINATION.md) — one postgres,
schema per strangler.

## §3d data model clarification (Ian, 2026-05-28)

**The parent forums in §3d (Repair and Restoration, New Builds, Tools/Spaces/Robots, Business, Market Place, Sponsor Forums) are NOT vestigial.** They're category-containers — discussion happens in their subforums, which are real active forums:

| Parent | Direct topics | Subforums | Sub-total topics | Sub-total replies |
|---|---|---|---|---|
| repair-and-restoration | 0 | 9 | 522 | 2172 |
| new-construction | 1 | 7 | 173 | 729 |
| tools-spaces-robots-and-widgets | 0 | 6 | 149 | 499 |
| business | 0 | 5 | 61 | 256 |
| market-place | 0 | 2 | 38 | 46 |
| sponsor-fourm ("Sponsor Forums") | 0 (forum_type=category) | 4 | 8 | 16 |

What §3d actually wants deleted at cutover: the **auto-enroll BB groups** (the `bp_groups_members` rows that auto-add ~1784 users to each). The forum-container rows themselves stay. Coordinator bug-bundle #6 conflated the two. **`HIDE_SLUGS` is now empty.**

**Render rule:** a top-level forum is treated as a non-clickable category header when (a) `forum_type='category'`, OR (b) `topic_count=0 AND has_subforums`. Discussion lives in subforums; the parent page would be empty. Edge: `new-construction` has 1 direct topic so it renders clickable — acceptable.

After this change `/forums-poc/` shows **5 category headers + 40 clickable forums** (25KB rendered).
