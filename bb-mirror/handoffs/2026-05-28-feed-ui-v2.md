# BB-mirror — Session Handoff (2026-05-28, feed-ui-v2)

## What this project is

Read-side strangler for BB/bbPress forum threads. Reads from postgres mirror
at native speed; writes round-trip through BB REST. Mu-plugin syncs WP→pg
in real time; systemd timer reconciles every 10 min.

### Feature 1: Sort bar

Three tabs above the feed: **New** (default) · **Old** · **Hot**.

- Rendered as `.feed-sort-bar` nav above the cards.
- Sort is read from `$_GET['sort']` (which `index.php` populates by parsing `REQUEST_URI` query string — nginx alias drops `$query_string` but leaves `REQUEST_URI` intact).
- **New**: `ORDER BY t.last_active_at DESC` (unchanged behavior, now explicit).
- **Old**: `ORDER BY t.last_active_at ASC`.
- **Hot**: `ORDER BY (reply_count::float / POW(age_hours + 2, 1.5)) DESC`.
- Sort links preserve the current `forum_slug` scope and reset `offset` to 0.

### Feature 2: Featured image thumbnail

`topic.featured_image_url` added to `SELECT`. If non-null/non-empty, renders a `.feed-card__thumb` image (88px wide, 4:3 aspect-ratio, object-fit: cover) on the right side of the card header. No thumbnail element rendered when null — no broken placeholders.

No topics currently have `featured_image_url` set in the mirror (column exists, all NULL). The UI is ready whenever upstream sync starts populating it.

### Feature 3: Card redesign — OP excerpt + threaded reply stubs

Old dual-card-kind design (`topic_start` / `reply_stack`) replaced with a single unified card for all topics.

**Query restructure:**
- Primary query now fetches topics directly (`FROM topic t JOIN forum f ...`) with the chosen sort.
- Second query fetches replies for all topic IDs in one round trip:
  `WHERE r.topic_id = ANY(:ids::bigint[]) ORDER BY r.topic_id, r.created_at DESC`.
- PHP groups replies by `topic_id` and slices: first 3 visible, remainder hidden behind accordion.

**Card anatomy:**
- Forum context + relative timestamp (top row, unchanged).
- Topic title (link to single-topic page).
- OP excerpt: `LEFT(content_text, 240)` → `strip_tags` → 200-char truncation + `…`.
- "Started by X · N time ago" meta line.
- Featured image thumbnail (right side, when present).
- Reply section: up to 3 reply stubs, each showing author (mahogany link if slug available), ~200-char excerpt, relative timestamp.
- If > 3 replies: expand button "show N more replies" that toggles `.replies-expanded` class → CSS reveals `.reply-stub--overflow` elements and hides the button.
- reply_count = 0: no reply section rendered at all.

**Join fix**: `forums.person` uses `id` (not `wp_user_id` — that column doesn't exist). Replied join is `JOIN person p ON p.id = r.author_id`. Person table has no `profile_url` column; author link is `/members/<slug>/` when `p.slug` is present.

## Gotchas discovered this session

1. **`forums.person` has no `wp_user_id` or `profile_url`**: uses `id` (= WP user ID) and `slug`. Author links point to `/members/<slug>/`. If slug is null (anonymous/deleted user), falls back to a `<span>` (no link).

2. **No topics have `featured_image_url` yet**: column exists and is selected, but all rows are NULL in the current mirror. The thumbnail section renders correctly when null (no img element). Feature is complete; will activate automatically when sync starts populating it.

3. **Hot sort on paginated deep offsets**: `ORDER BY` on computed score + `LIMIT/OFFSET` means page 2 is consistent only within a request — not cursor-stable. Acceptable for v0.

4. **`index.php` already handles `?sort=`**: the upstream `index.php` re-parses `REQUEST_URI` into `$_GET` before including `_feed.php`, so `$_GET['sort']` is populated correctly even under the nginx alias that strips `$query_string`.

5. **Old reply-stack JS replaced**: the prior `aria-controls`/`hidden` toggle in section 2 of `forums.js` is superseded by the new CSS-class accordion. The old code is gone; the new code is 3 lines.

6. **Prior gotchas still apply**: nginx alias drops query string, dual `acoustic` slug, file ownership pattern — see `handoffs/2026-05-28-activity-feed-ia.md` and earlier.
