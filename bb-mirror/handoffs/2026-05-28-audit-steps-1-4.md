### Why some Local Looths aren't visible on the public forum list

7 of the 9 regional groups are `private` (SoCal, DMV, Tri-State NYC, etc.) — their forums have `post_status='private'`, our `derive_visibility()` maps that to `visibility='private'`, and the forum-list query filters to `visibility='public'`. They have correct group_id + effective_group_id data — they just don't surface to non-members.

Group-member-aware private visibility is a future feature behind `/whoami` + a `user ↔ group` membership table (profile-app owns the latter post-cutover). Until then, private group forums are visible only to admins via direct URL (no listing).

## Orphan-gate rule (codified)

A forum whose `effective_group_id` references a deleted bp_group (NULL after LEFT JOIN) falls back to ungated — visible + postable to all authenticated viewers. The render path implements this via LEFT JOIN; reply-form gating (later session) will too.

This matches the cutover-day scenario in §3d: when the 5 auto-enroll groups are deleted, their ~27 subforums stay readable/postable. No regression.

## Notes / gotchas

- `wp_bp_groups_groupmeta.forum_id` is a **serialized PHP array** (`a:1:{i:0;i:3818;}`), not a plain int. Backfill + sync both deserialize via `first_group_id_from_meta()`. SQL CASTs return 0 for these — don't try that.
- `_bbp_group_ids` postmeta is also serialized. Same deserializer reused.
- `effective_group_id` is NOT maintained perfectly by per-row sync. Per-forum upsert refreshes the affected forum + descendants, but if a forum's `group_id` changes such that an UNRELATED subtree should now have a different effective_group_id, only a backfill or reconcile cron catches it. Acceptable for v0; reconcile cron handles this.
- 7 regional Looths forums are correctly mirrored but invisible on public forum-list (see "Why some Local Looths aren't visible" above).
- `person.is_moderator` still all false (data-side fill is a separate work item).
- SQLite fallback paths remain in `config.php`, `bin/init-db.php`, `bin/backfill.php`, `api/v0/_sync.php`. Rollback window not closed.
