### 4. Site-wide header (cloned BB)

[templates/partials/site-header.php](templates/partials/site-header.php) + [assets/lg-site-header.css](assets/lg-site-header.css) + [.js](assets/lg-site-header.js). Standalone CSS, inline SVG icons (no `bb-icons` font dependency). Uses BB's existing nav-menu locations (`header-menu`, `header-menu-logout`, `mobile-menu-logged-in/out`). Currently included by [templates/single-v2.php](templates/single-v2.php) — **only on v2-managed posts**.

User flagged this as the wrong scope on the last turn — wants the same header **site-wide** via `buddyboss-theme-child-1.0.0/header.php` override. **NOT YET DONE.** See open items.

Behaviors:
- **Fixed** position (switched from `sticky` — BB CSS layer was killing it). Body gets `padding-top: 64px`. Offsets for WP admin bar.
- **Priority-nav**: items overflowing the row collapse into a "···" dropdown (`layout()` in JS, re-runs on resize).
- **Compact-on-scroll**: at `scrollY > 60`, translucent blurred backdrop + soft shadow. NO size changes (earlier attempts at shrinking heights/widths felt "shrunken-together" and were reverted).
- **Hamburger**: far-left on mobile (`≤960px`), hides desktop nav.
- **Search button** → inline search panel.
- **Avatar** = direct link to BP profile-edit (no dropdown — earlier dropdown was stripped per user ask).
- **Badge counts lazy-loaded** by JS (server skips BP queries on initial render). Fetches `/buddyboss/v1/messages` + `/notifications` ~200ms after load, then every 60s while tab visible. **Caveat**: relies on `X-WP-Total` / `X-BP-Unread-Count` response headers — needs verification in production.

### 5. Strat article (post 69369) rewrite

Rewritten twice under different image-aspect rules. Current rule: **aspect (`width/height`) < 0.87 = "long" paired**, else single. Orphan longs pair with their caption as text in column 2. Wide images render full-bleed single with native figcaption.

Final layout (15 blocks): post-header → editor-note callout → 6 columns rows → 3 singles → divider → outro wysiwyg → references callout (data variant, 5 links from web search on Callaham/Fender/Gruhn) → post-footer.

### 6. Architecture / cascade additions

- **Article container responsive** ([src/CssBuilder.php](src/CssBuilder.php) + [src/theme/tokens.json](src/theme/tokens.json)):
  - `--lg-article-max` → `clamp(20rem, 88vw, 1040px)` (was static 720px)
  - `--lg-article-padding-inline` → `clamp(16px, 4vw, 48px)`
  - `--lg-article-max-wide` (1320px) for opt-in wider blocks
  - Full-bleed escape: `.lg-article > .lg-post-header, .lg-article > .lg-post-footer, .lg-article > .lg-fullbleed { width: 100vw; margin-inline: calc(50% - 50vw); }`
- **Renderer threads `path`** through `renderBlock` and `renderChildren`, emits `data-lg-block-path` on `<lg-edit>` markers. Markers now emit on either `editor_mode` OR `can_edit` (the latter for link-edit pencil to locate blocks without full edit mode).
- **Image block wraps the `<img>` in `.lg-image__frame`** so border-radius + zoom are bounded together. Photo sub_target now targets `.lg-image__frame` (was `.lg-image__img` — caused variant/dash radius edits to collide).

### 7. Dash + metabox fixes

- **Boolean props** render as real checkboxes with hidden `value="0"` companion ([src/MetaBox.php](src/MetaBox.php) `render_scalar_field`). Earlier they rendered as text inputs which cast unchecked → false → silently disabled `show_author`/`show_related` on the Strat post.
- **Dash `sanitize_group`**: removed the "drop value if matches default" rule ([src/Dash.php](src/Dash.php)). It was eliminating explicit T-button transparent saves. Always-store now.
- **Schema doc generator** ([bin/generate-schema-doc.php](bin/generate-schema-doc.php)) now dumps per-variant CSS-var overrides so authors see the actual shape, not just the variant key.

## Gotchas the next session will trip over

- **Editor markers only emit when `editor_mode` OR `can_edit`** — so anon HTML stays clean (cached), but admin/author renders include `<lg-edit>` markers always (used by the link-edit pencil to locate blocks). This is why anon and admin renders differ in size.
- **`get_the_title()` returns entity-safe HTML** (curly quotes etc as `&#8217;`). Don't `htmlspecialchars()` it again — that double-escapes. Echo directly. Applied to title + tagline in post-header and card-title in post-footer.
- **Boolean props need real checkboxes** in the metabox (fixed) — and `array_key_exists`-style defaults in render.php (`!empty($args[...])` treats unset as false even when manifest default is true). See [blocks/post-footer/render.php](blocks/post-footer/render.php) for the pattern.
- **Image variant + dash photo sub_target both target `.lg-image__frame`** now — not the inner `<img>`. The frame is the chrome owner since the zoom-clipping wrap was added.
- **Variant pill suppresses `editor.variant_options`** keys NOT in the allowlist — used by `section-heading` to hide its h2/h3/h4 from the pill (those are level-driven, level lives on the metabox dropdown).
- **`isColumnChild(path)`** in the FE editor JS = `path.length >= 5 && path[length-2] === 'blocks'`. Used to decide whether to render left/right arrow pills on a block.
- **Site header is currently v2-only** (in `templates/single-v2.php`). Other pages still render BB's native header. This is open item #1.
