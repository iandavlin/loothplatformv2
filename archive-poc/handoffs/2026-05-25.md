# archive-poc — Session Handoff (2026-05-25)

## What this project is

A standalone discovery front for loothgroup.com that **replaces the BB activity feed** at `/archive-poc/`. Owns its own PHP-FPM pool, reads from a SQLite index (`/srv/archive-poc/index.sqlite` on live), and surfaces content via two-pane layouts + carousel rows.

**Code lives:**
- Source: `/home/ubuntu/projects/archive-poc/` (this dir, on dev box)
- Deploys to: `/srv/archive-poc/` on live (54.157.13.77)
- Mu-plugin source: `/var/www/dev/wp-content/mu-plugins/archive-poc-sync.php`
- Mu-plugin deploys to: `/var/www/html/wp-content/mu-plugins/archive-poc-sync.php` on live

**Sockets / pools:**
- archive-poc FPM pool: `/run/php/php8.3-fpm-archive-poc.sock`
- looth-live FPM pool: `/run/php/php8.3-fpm-looth-live.sock` (handles WP + mu-plugin)

## Current state of the front page

`/archive-poc/` renders a **3-pane "activity band"** at the top, then traditional carousel rows below.

```
┌──────────────────────────────────────────────────────────────┐
│  ┌────────┐  ┌──────────────────────┐  ┌─────────────────┐   │
│  │ left   │  │ center               │  │ right           │   │
│  │ pane   │  │ activity feed        │  │ sponsors        │   │
│  │        │  │ vertical scroll      │  │ scroll if over  │   │
│  │  CTAs  │  │ stack of cards       │  │  ★ tile/logo    │   │
│  │ Events │  │                      │  │                 │   │
│  └────────┘  └──────────────────────┘  └─────────────────┘   │
└──────────────────────────────────────────────────────────────┘
  ↓ below the band: normal page flow
    - New this week
    - Active discussions
    - Most-liked of all time
    - Latest videos
    - tag-random rows
```

**Sidebar content** (left + right) comes from PHP constants at the top of `web/index.php`: `LG_SPONSORS`, `LG_LOCAL_LOOTHS`, `LG_CTA_MEMBER`, `LG_CTA_PUBLIC`. Edit those arrays → redeploy `index.php` → done. No DB, no admin UI.

**Breakpoints:**
- ≥1100px: full 3-pane
- 760–1100px: drops left CTA rail, keeps main + right
- ≤760px: all stack vertically; activity reverts to horizontal swipe rail

Body scroll is **not** locked (an app-shell attempt was abandoned). Page scrolls normally; the band is a 640px-tall section.

## Search modal

CTA **Search the Archive** in left sidebar opens a modal (HTML in index.php near footer). 180ms-debounced fetch to `/archive-api/v0/search?q=...&limit=3` → top-3 hits → **See all N results →** button hands the query off to the page's `#q` input which triggers the existing grid-mode search.

## Activity REST endpoint

`/wp-json/looth/v1/activity` — cached 30s in Redis db0 (key: `lg_act_<m|p>_<before>_<limit>`).

BB whitelist for `/wp-json/looth/v1/*` was added in WP-admin (BuddyBoss → Private REST API). Don't disable.

Activity types ingested (`LG_ACTIVITY_TYPES`):
- `new_blog_*` for each CPT (including `new_blog_event` — added this session)
- `bbp_topic_create`
- `new_member`
- `activity_update`

**Tier gating:** items whose post has tier `looth-lite`/`looth-pro` are dropped for **anonymous** visitors. Events tagged Lite/Pro only show up to logged-in users — this is correct, not a bug. Anonymous testing in incognito shows zero of those.

## BB reactions

Schema: `wp_bb_user_reactions` (user_id, reaction_id, item_type, item_id) + `bb_reaction` CPT (post_content = serialized array with `icon` = unicode hex codepoint for "emotions" type, or `custom/<hash>` for image icons).

Mu-plugin has `lg_activity_reactions($activity_id)` + `lg_activity_decode_reaction()`. Returns `['total' => N, 'top' => [{emoji, label, count} × 5]]`.

**Custom (image-based) reactions fall back to ★** because we don't ship the asset URL. Lots of "Thumbs Up" reactions on live are custom — they show as ★ with the label readable on hover. Improvement path: pull the custom icon attachment URL via `_thumbnail_id` on the reaction post.

## Image pipeline (the long saga)

Resolution chain (same logic in indexer.php, backfill.php, mu-plugin):
1. `wp_get_attachment_image_url($tid, 'full')` if featured image exists — originals
2. `bp_media_ids` postmeta → first `wp_bp_media.attachment_id` (BB media attachments on topics)
3. First `<img src>` regex from `post_content`
4. Frontend `onerror=` swaps to `LG_FALLBACK_IMG`

**Notes:**
- `LG_FALLBACK_IMG = 'https://loothgroup.com/wp-content/uploads/2024/11/Featured-Image-Fallback-2.webp'`
- `ewww_image_optimizer_maxmediawidth = 1200` caps **new** uploads at 1200px wide. Old uploads pre-EWWW are larger.
- An R2 graveyard guard was REMOVED from indexer.php this session — it was nulling perfectly good URLs on live (where R2 serves files not synced to local disk).

## Events (ACF `event` CPT)

ACF fields read by indexer / backfill / mu-plugin:
- `events_start_date_and_time_` (Ymd e.g. `20260528`)
- `time_of_event` (HH:MM:SS)
- `region` (term ID → wp_terms lookup for name)
- `zoom_url_for_looth_group_virtual_event`
- `event_tier_` (tier taxonomy, gates visibility)

Event card link in sidebar = post permalink, **not** Zoom URL (don't leak gated content to anon). The post page handles tier-gating for the Zoom link.

ICS calendar button in event-card foot opens dropdown: Google / Apple / Outlook / `.ics` download. Implemented in archive.js for archive-poc, and also in the standalone `[looth_events]` shortcode mu-plugin (`/home/ubuntu/projects/lg-events-shortcode.php`).

After mu-plugin changes: `sudo redis-cli -n 0 FLUSHDB` to drop the 30s activity cache. **NOT** `FLUSHALL` — db2 is loothtool.com's cache.
