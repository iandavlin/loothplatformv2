Path 2 — User searches the archive

type a query on /archive-poc/ or /archive/

Search is a client-side fetch to the archive-poc JSON API, answered by a SQLite FTS5 full-text query. No WordPress, no SQL LIKE table-scan — a Porter-stemmed FTS index with BM25 ranking and faceting.

Browser · JS
archive.js → fetch()
GET /archive-api/v0/search?q=mandolin&kind=article&tier=public&sort=relevance
Query state mirrors the URL (?q=&kind=&tag=&author_id=&sort=&page=); credentials: same-origin.
↓
nginx · cookie gate
location ^~ /archive-api/v0/
rewrite /search → search.php · requires $loothdev_is_authorized (search is public-read)
↓ fastcgi_pass
FPM pool
php8.3-fpm-archive-poc.sock
↓
App · API
archive-poc/api/v0/search.php
parse q/filters → fts_quote(q) → build WHERE → PDO query (query_only=ON)
↓ FTS5 MATCH
Database · SQLite
index.sqlite · content_item ⋈ content_fts
… JOIN content_fts f ON f.rowid=ci.id WHERE content_fts MATCH ?  ORDER BY bm25(content_fts)
+ a second pass over the unlimited result set to build facet counts (kind / tier / tag / author); tags bulk-loaded from content_tag ⋈ tag.
↓
Render
JSON { items[], facets{}, total } → JS paints grid + facet rail
each item.url → /article/<slug>/ etc. (continues into Path 4)
